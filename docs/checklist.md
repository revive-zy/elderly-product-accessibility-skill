# 适老化研发验收清单

## 页面结构

- 页面唯一主任务是否明确?
- 用户 3 秒内能否知道下一步点哪里?
- 首页核心入口是否不超过 5 个?
- 页面是否按任务组织，而不是按业务频道组织?
- 是否提供明确的返回、取消、修改或继续路径?

## 交互

- 主按钮是否足够大，且不与危险按钮紧贴?
- 核心操作是否没有依赖长按、滑动、双指、三指、悬停或隐藏菜单?
- 是否没有非必要倒计时、自动跳转、随机弹窗?
- 弹窗关闭按钮是否明显且命中区域足够大?
- 点击、加载、成功、失败是否都有明确反馈?

## 信息展示

- 主要文字是否不小于 18 dp/pt?
- 正文是否建议在 18 到 22 sp/pt?
- 核心数字、标题、按钮是否建议在 22 到 28 sp/pt?
- 字号在 1.5x 和 2x 下是否不截断、不遮挡、不重叠?
- 普通文本对比度是否达到 4.5:1?
- 大字号文本对比度是否达到 3:1?
- 是否没有只用颜色表达状态?
- 图标是否配文字?

## 表单和错误

- 表单是否一页只处理一类信息?
- 字段是否只收集必要信息?
- 字段是否说明为什么需要?
- 手机号、身份证、银行卡、日期、地址是否自动格式化?
- 验证码是否提供语音或替代方式?
- 错误提示是否说明原因和解决办法?

## 安全和信任

- 是否没有广告插件、随机广告或临时广告弹窗?
- 是否没有诱导下载、诱导付款或欺骗性按钮?
- 权限和个人信息收集是否最小必要?
- 支付、授权、删除、法律承诺等高风险动作是否有确认页?
- 是否提供凭证、撤销、客服或家人协助?

## 辅助技术

- Android 是否通过 TalkBack 测试核心路径?
- iOS 是否通过 VoiceOver 测试核心路径?
- Web 是否支持键盘导航和可见焦点?
- 非文本控件是否有清晰语义描述?
- 纯装饰图片是否不干扰读屏?
