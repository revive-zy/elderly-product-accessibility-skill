# Audit Report Example: 旧产品适老化改造

## P0 问题

- 首页入口过多，用户无法判断下一步。
- 支付确认页未突出金额、周期和取消方式。
- 验证码失败只提示"参数错误"。
- 客服入口隐藏在二级页面。

## P0 改造

- 首页保留 3 到 5 个核心任务入口。
- 支付页突出金额、收款方、周期、取消方式和客服。
- 验证码错误改为可理解提示，并提供语音验证码。
- 在首页、支付页、失败页增加客服和家人协助入口。

## P1 问题

- 字号跟随系统放大后按钮文字截断。
- 列表项标签过多。
- 错误态没有下一步动作。

## P1 改造

- 调整组件高度和换行策略。
- 列表项只保留名称、状态、关键日期和下一步动作。
- 所有错误态增加原因、重试、修改和客服入口。

## 验收

- TalkBack/VoiceOver 可读出核心按钮含义。
- 1.5x 和 2x 字体下无遮挡。
- 主按钮点击区域不小于 60 x 60 dp/pt。
- 普通文本对比度不低于 4.5:1。
